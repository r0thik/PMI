## Программа и методика испытаний

## 1. Объект испытаний

**Наименование:** Веб-приложение Интернет-магазин игр «Jinki» на базе Flask.

**Область применения:** Тестирование безопасности учебного веб-ресурса, представляющего собой интернет-каталог с функциями регистрации, аутентификации, просмотра товаров, новостей и административной панелью.

**Краткая характеристика:**  
- Бэкенд: Python 3.12 + Flask.  
- База данных: PostgreSQL.  
- Контейнеризация: Docker, Docker Compose.  
- Frontend: HTML, CSS.  

**Основные функциональные модули:**  
- Публичная часть: главная страница (`/`), каталог (`/catalog`), аутентификация (`/login`, `/register`).  
- Авторизованная часть: профиль, покупка товаров (`/buy/...`).  
- Административная часть: панель управления (`/admin`).  

**Технологический стек для целей испытаний:**  
- Протокол: HTTP/1.1.  
- Сервер: Werkzeug.  
- Клиентская часть: браузер с поддержкой HTML/CSS.  
- База данных: PostgreSQL 15+.

## 2. Цель проведения испытаний
Оценка уровня защищенности веб-приложения и выявление потенциальных уязвимостей, влияющих на КЦД.

## 3. Общие положения

**Основание для проведения испытаний:**  
Учебная лабораторная работа по дисциплине «Анализ защищённости веб-ресурсов».

**Продолжительность испытаний:**  
Продолжительность, время и место проведения испытаний определяются самостоятельно в зависимости от технических возможностей и условий проведения тестирования. Испытания проводятся в локальной среде с использованием персонального компьютера.

**Место проведения испытаний:**  
Локальная среда разработки с развёрнутым приложением через Docker Compose. Доступ к сети Интернет не требуется, кроме загрузки инструментов.

**Документы, используемые при испытаниях:**  
- OWASP WSTG (версия 4.2) – русскоязычный перечень тестов.  
- OWASP ASVS (уровень 1 – базовый).  
- Руководство по использованию инструментов: Burp Suite Community, ZAP, sqlmap, dirb, ffuf, Selenium.  

**Участвующие организации и лица:**  
- Исполнитель: студент.  
- Проверяющая сторона: преподаватель.
